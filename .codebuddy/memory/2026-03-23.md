# 2026-03-23 工作日志

## 创建了 C99 编译器工程 (c99lang)

### 项目概述
从零搭建了一个完整的 C99 编译器工程，命名为 `c99c`，使用 C11 编写，CMake 构建系统。

### 架构设计
采用经典的多阶段编译流水线：
1. **预处理器** → 2. **词法分析** → 3. **语法分析(AST)** → 4. **语义分析** → 5. **IR生成(三地址码)** → 6. **x86-64代码生成**

### 已实现的模块
- **工具库**: Arena内存池、字符串驻留表(strtab)、动态数组(vector)、错误诊断
- **预处理器**: #include/#define/#undef/#if/#ifdef/#ifndef/#elif/#else/#endif, 宏展开, 内建宏(__LINE__/__FILE__/__DATE__)
- **词法分析器**: 完整支持C99所有token类型(37个关键字, 45+运算符, 整数/浮点/字符/字符串字面量, C99十六进制浮点)
- **语法分析器**: 递归下降解析器, 支持声明、语句、表达式(运算符优先级攀升法), C99 for循环声明、designated initializer、compound literal等
- **语义分析**: 类型检查、符号表管理、隐式转换(integer promotion, usual arithmetic conversions)、作用域管理
- **IR生成**: 三地址码中间表示, 支持所有控制流(if/while/for/do-while/break/continue), 短路求值
- **x86-64代码生成**: AT&T语法汇编输出, System V ABI, 函数序言/尾声, 栈帧管理

### 测试结果 (扩展后)
10 个测试套件，全部通过：
1. **test_arena** (10156 assertions) - Arena内存池：init/free、alloc、calloc、strdup、strndup、大分配、混合分配
2. **test_vector** (35 assertions) - 动态数组：push/pop/get/set/clear、增长、NULL、序列操作
3. **test_strtab** (562 assertions) - 字符串表：interning、去重、子串、空串、大量字符串、C99关键字
4. **test_lexer** (62 assertions) - 词法分析器原始测试
5. **test_lexer_extended** (181 assertions) - 词法分析器扩展：字符串/字符字面量、所有关键字、运算符、十六进制/八进制、浮点、后缀、行号跟踪、peek/match、边界情况
6. **test_preprocessor** (54 assertions) - 预处理器：#define/#undef/#ifdef/#ifndef/#if/#elif/#else、嵌套条件编译、函数宏、注释删除、字符串透传
7. **test_parser** (23 assertions) - 语法分析器原始测试
8. **test_parser_extended** (67 assertions) - 语法分析器扩展：while/do-while/switch/goto、指针/数组/enum/union、三元/cast/sizeof、复合赋值、函数调用、成员访问、变参函数等30个测试
9. **test_type_symtab** (98 assertions) - 类型系统：单例类型、指针/数组/函数类型构造、is_integer/floating/arithmetic/scalar/complete/compatible查询；符号表：insert/lookup、作用域push/pop、嵌套作用域、枚举常量
10. **test_ir** (147 assertions) - IR模块：module/func/block/instr创建、IRVal构造、完整函数构建、op_str、全部42种操作码

同时修复了预处理器 #ifdef 的 bug（pp_is_active 在 if_depth 增加后调用导致读到未初始化值）。
